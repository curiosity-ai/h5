# Attributes Reference

Attributes are the primary way to control how C# code is translated into JavaScript in H5. Most attributes are found in the `H5` namespace.

## General Purpose Attributes

### `[External]`
Indicates that the type or member is defined in external JavaScript code and should not be generated by the compiler.
```csharp
[External]
public class MyExternalLibrary {
    public void DoSomething() { }
}
```

### `[Name]`
Specifies the name to use for the entity in the generated JavaScript.
```csharp
[Name("myFunction")]
public void CSharpFunction() { }
```

### `[Namespace]`
Specifies a custom namespace for the generated JavaScript entity.
```csharp
[Namespace("MyCompany.Utils")]
public class MyClass { }
```

## Method and Logic Control

### `[Template]`
Replaces a method call with a custom JavaScript code template. Useful for mapping C# methods to specific JavaScript expressions.
```csharp
[Template("console.log({message})")]
public static void Log(string message) { }
```

### `[Script]`
Discards the C# implementation and uses the provided JavaScript lines for the method body.
```csharp
[Script("return a + b;")]
public int Add(int a, int b) { return 0; }
```

## Type System Control

### `[ObjectLiteral]`
Tells the compiler to treat the class or struct as a plain JavaScript object literal `{}` instead of creating a full-blown H5 class.
```csharp
[ObjectLiteral]
public class Config {
    public string Host { get; set; }
    public int Port { get; set; }
}
```

### `[Enum]`
Controls how enums are emitted.
- `Emit.Name`: Emits the name of the enum member.
- `Emit.Value`: Emits the underlying integer value.
- `Emit.StringName`: Emits the name as a string.

## Modules

### `[Module]`
Specifies that the type or assembly should be emitted as a module (AMD, CommonJS, ES6, etc.).
```csharp
[Module(ModuleType.ES6)]
public class MyModule { }
```
